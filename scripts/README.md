# Scripts Directory

This directory contains utility scripts for maintaining the BambooHR API Python SDK.

## post_generate.py

**Purpose:** Runs the full post-generation pipeline after `openapi-generator` outputs to the project root.

The generator runs with default output (project root), placing package code at `bamboohr_sdk/`,
docs at `docs/`, etc. This script handles cleanup and processing after generation.

**Steps:**

1. Cleans up unwanted generator artifacts (`.gitlab-ci.yml`, `.github/`, etc.)
2. Removes `PublicAPIApi` files and references (internal-only API)
3. Formats generated code with `ruff`
4. Adds custom headers to API documentation
5. Cleans up obsolete files (compares old vs new FILES manifest)

**Usage:** Called automatically by `make generate`. Not typically run manually.

```bash
python scripts/post_generate.py
```

## cleanup_obsolete_files.py

**Purpose:** Removes files that are no longer generated by the OpenAPI Generator.

**How it works:**

1. Before generation, the Makefile backs up `.openapi-generator/FILES` to `.openapi-generator/FILES.old`
2. After generation, this script compares the old and new FILES manifests
3. Files in the old manifest but not the new one are considered obsolete
4. The script prompts for confirmation before deleting (unless `--force` is used)
5. Empty directories are cleaned up automatically

**Usage:**

```bash
# Automatic (runs as part of make generate with --force)
make generate

# Manual with confirmation prompt
make cleanup-obsolete
# or directly:
python scripts/cleanup_obsolete_files.py

# Force mode (no confirmation, for CI/unattended runs)
python scripts/cleanup_obsolete_files.py --force
```

**First Run:** On the first run there will be no `.openapi-generator/FILES.old` backup,
so the script will skip cleanup. This is expected. Subsequent runs will properly detect
and remove obsolete files.
